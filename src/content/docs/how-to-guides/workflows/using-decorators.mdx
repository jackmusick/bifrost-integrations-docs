## Overview

Bifrost uses three decorators to define workflows:
- `@workflow` - Registers executable workflow
- `@param` - Defines input parameters
- `@data_provider` - Provides dynamic options

## @workflow Advanced Options

### Scheduling Workflows

Run workflows automatically on a schedule:

```python
@workflow(
    name="daily_cleanup",
    description="Daily cleanup task",
    execution_mode="scheduled",
    schedule="0 2 * * *",  # Daily at 2 AM UTC
    expose_in_forms=False   # Hide from manual execution
)
async def daily_cleanup(context):
    # Runs automatically
    pass
```

**Cron Expressions**:
- `0 9 * * *` - Daily at 9 AM
- `0 */6 * * *` - Every 6 hours
- `0 9 * * 1` - Every Monday at 9 AM
- `0 0 1 * *` - First day of month at midnight

### Retry Policies

Automatically retry failed workflows:

```python
@workflow(
    name="api_call",
    description="Call external API with retries",
    retry_policy={
        "max_attempts": 3,
        "backoff_seconds": 5
    }
)
async def api_call(context):
    # Will retry up to 3 times with 5 second delay
    pass
```

### Custom Permissions

Require specific permissions to execute:

```python
@workflow(
    name="admin_task",
    description="Admin-only workflow",
    required_permission="canManageOrganization"
)
async def admin_task(context):
    # Only users with canManageOrganization can run
    pass
```

## @param Advanced Patterns

### Dependent Parameters

Use data providers with parameters:

```python
@data_provider(name="get_users_by_dept")
@param("department", type="string", required=True)
async def get_users_by_dept(context, department: str):
    # Fetch users for specific department
    users = await fetch_users(department)
    return [{"label": u.name, "value": u.id} for u in users]

@workflow(name="assign_to_user")
@param("department", type="string", data_provider="get_departments")
@param("user", type="string", data_provider="get_users_by_dept")
async def assign_to_user(context, department: str, user: str):
    # User dropdown populates based on selected department
    pass
```

### Complex Validation

Combine multiple validation rules:

```python
@param(
    "password",
    type="string",
    required=True,
    validation={
        "min_length": 12,
        "max_length": 128,
        "pattern": r"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]+$"
    },
    help_text="Min 12 chars, must include uppercase, lowercase, number, and special char"
)
```

### Conditional Defaults

Set defaults based on context:

```python
@param(
    "assignee",
    type="string",
    default_value=None  # Set in workflow logic
)
async def create_ticket(context, assignee: str = None):
    # Default to current user if not specified
    if not assignee:
        assignee = context.user_id
    pass
```

## @data_provider Advanced Patterns

### Caching Strategies

Control cache behavior:

```python
@data_provider(
    name="get_departments",
    cache_ttl_seconds=3600  # Cache for 1 hour
)
async def get_departments(context):
    # Expensive query, cache results
    return await fetch_departments()

@data_provider(
    name="get_current_user",
    cache_ttl_seconds=0  # Never cache (always fresh)
)
async def get_current_user(context):
    # Always fetch latest user data
    return await fetch_user(context.user_id)
```

### Hierarchical Options

Create nested/grouped options:

```python
@data_provider(name="get_apps_by_category")
async def get_apps_by_category(context):
    return [
        {
            "label": "Microsoft 365",
            "value": "m365",
            "metadata": {
                "group": "Productivity",
                "icon": "microsoft"
            }
        },
        {
            "label": "Google Workspace",
            "value": "google",
            "metadata": {
                "group": "Productivity",
                "icon": "google"
            }
        },
        {
            "label": "Slack",
            "value": "slack",
            "metadata": {
                "group": "Communication",
                "icon": "slack"
            }
        }
    ]
```

### Dynamic Filtering

Filter options based on user/org:

```python
@data_provider(name="get_available_licenses")
async def get_available_licenses(context):
    # Only show licenses available to this org
    licenses = await fetch_org_licenses(context.org_id)

    # Filter by what user can assign
    if not context.is_platform_admin:
        licenses = [l for l in licenses if not l.requires_admin]

    return [
        {"label": l.name, "value": l.sku}
        for l in licenses
    ]
```

## Decorator Order Matters

Parameters are applied bottom-to-top:

```python
@workflow(name="example")
@param("third", type="string")   # Shows FIRST in form
@param("second", type="string")  # Shows SECOND
@param("first", type="string")   # Shows LAST
async def example(context, first, second, third):
    pass
```

To match declaration order, reverse the decorators:

```python
@workflow(name="example")
@param("first", type="string")   # Shows FIRST in form
@param("second", type="string")  # Shows SECOND
@param("third", type="string")   # Shows THIRD
async def example(context, first, second, third):
    pass
```

## Best Practices

1. **Use Descriptive Names**: `create_user_in_m365` not `proc1`
2. **Set Appropriate Timeouts**: Match expected execution time
3. **Cache Wisely**: Balance freshness vs. performance
4. **Validate Early**: Use `@param` validation before workflow runs
5. **Document Parameters**: Use `help_text` for user guidance
6. **Test Data Providers**: Verify they return correct format
7. **Use Retry Policies**: For unreliable external APIs
8. **Schedule Appropriately**: Use UTC times for schedules

## Common Pitfalls

### ❌ Missing Required Parameters
```python
@workflow()  # Missing name and description
```

### ✅ Always Provide Required
```python
@workflow(name="example", description="Example workflow")
```

### ❌ Wrong Parameter Type
```python
@param("email", type="text")  # No such type
```

### ✅ Use Valid Types
```python
@param("email", type="email")  # Valid type with validation
```

### ❌ Sync Mode for Long Tasks
```python
@workflow(execution_mode="sync", timeout_seconds=1800)
async def bulk_import():  # Will timeout/block users
    pass
```

### ✅ Async for Long Tasks
```python
@workflow(execution_mode="async", timeout_seconds=1800)
async def bulk_import():  # Runs in background
    pass
```
