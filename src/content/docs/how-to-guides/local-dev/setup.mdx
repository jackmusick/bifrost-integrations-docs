## Prerequisites

- **Docker** and **Docker Compose** installed
- **VS Code** with Python extension (for debugging)

## Quick Start

<Steps>

1. **Clone the Bifrost repository**:

    ```bash
    git clone https://github.com/jackmusick/bifrost.git
    cd bifrost
    ```

2. **Run setup script**:

    ```bash
    ./setup.sh
    ```

3. **Start all services**:

    ```bash
    docker compose up
    ```

4. **Access the application** at `http://localhost:3000`

    The first user to register becomes the Platform Admin.

</Steps>

## Development with Hot Reload

For active development with hot reload on code changes:

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```

### What the Dev Overlay Adds

The `docker-compose.dev.yml` overlay adds:

| Service | Dev Behavior |
|---------|--------------|
| **api** | Hot reload with uvicorn `--reload`, debugpy support |
| **worker** | Auto-restart on code changes via watchmedo |
| **scheduler** | Auto-restart on code changes |
| **client** | Vite dev server with HMR |

## Debugging with VS Code

<Steps>

1. **Enable debug mode** in `.env`:

    ```bash
    BIFROST_DEBUG=true
    ```

2. **Add a launch configuration** (`.vscode/launch.json`):

    ```json
    {
      "version": "0.2.0",
      "configurations": [
        {
          "name": "Attach to Bifrost API",
          "type": "debugpy",
          "request": "attach",
          "connect": {
            "host": "localhost",
            "port": 5678
          },
          "pathMappings": [
            {
              "localRoot": "${workspaceFolder}/api/src",
              "remoteRoot": "/app/src"
            },
            {
              "localRoot": "${workspaceFolder}/api/shared",
              "remoteRoot": "/app/shared"
            }
          ],
          "justMyCode": false
        }
      ]
    }
    ```

3. **Start with dev overlay** and attach debugger (F5)

</Steps>

<Aside type="note">
When debug mode is enabled, check API logs for "Waiting for debugger..." message.
</Aside>

## Creating Workflows

Create Python files in the UI editor or upload via the API. They're stored in S3 and synced to workers automatically.

```python
from bifrost import workflow, context

@workflow
async def hello_world(name: str):
    """A simple example workflow."""
    org = context.org_id
    return {"message": f"Hello, {name}!", "org": org}
```

See the [Writing Workflows](/how-to-guides/workflows/writing-workflows/) guide for more details.

## Viewing Logs

```bash
# All services
docker compose logs -f

# Specific service
docker compose logs -f api
docker compose logs -f worker
docker compose logs -f scheduler
```

## Stopping the Environment

```bash
docker compose down
```

To also remove volumes (fresh database):

```bash
docker compose down -v
```

## Troubleshooting

### Port Conflicts

Check if ports are in use:

```bash
lsof -i :3000   # Client
lsof -i :5678   # Debugpy
lsof -i :5432   # PostgreSQL
lsof -i :9000   # MinIO
```

### Debugger Won't Attach

1. Verify `BIFROST_DEBUG=true` in `.env`
2. Check API logs for debugpy message:
   ```bash
   docker compose logs api | grep debugpy
   ```
3. Restart the environment

### Workflows Not Discovered

1. Check API logs for import errors:
   ```bash
   docker compose logs api | grep -i error
   ```
2. Ensure Python files have valid `@workflow` decorators
3. Verify there are no syntax errors in your workflow files
