Bifrost stores secrets in Azure Key Vault with automatic org-scoping and encryption.

## Store Secrets

### Via UI

1. **Settings** → **Secrets** → **+ Add Secret**
2. Name: `api_key`
3. Value: `sk_live_xxxxx`
4. Click **Save**

### Via Config

Store as config with `secret_ref` type:

1. **Settings** → **Configuration** → **+ Add Config**
2. Key: `api_key`
3. Type: **Secret Reference**
4. Value: `api_key` (name of secret in Key Vault)

## Use in Workflows

```python
from bifrost import secrets, config

@workflow(name="call_api")
async def call_api(context):
    # Direct secret access
    api_key = secrets.get("api_key")

    # Or via config (if configured as secret_ref)
    api_key = config.get("api_key")  # Auto-resolves from Key Vault

    headers = {"Authorization": f"Bearer {api_key}"}
    # Make API call...
```

## Organization Scoping

Secrets are automatically scoped to organizations:

```python
# Stored in Key Vault as: {org_id}--api_key
secrets.set("api_key", "sk_live_123")

# Retrieved for current org only
api_key = secrets.get("api_key")
```

Each organization has completely isolated secrets.

## Common Use Cases

### API Keys
```python
@workflow(name="external_api")
async def external_api(context):
    api_key = secrets.get("external_api_key")
    # Use in API calls
```

### Database Credentials
```python
@workflow(name="db_query")
async def db_query(context):
    db_password = secrets.get("database_password")
    conn = await create_connection(
        host="db.example.com",
        password=db_password
    )
```

### Webhooks
```python
@workflow(name="webhook_verify")
async def webhook_verify(context, signature: str):
    webhook_secret = secrets.get("webhook_secret")
    # Verify signature
```

## Best Practices

1. **Never Hardcode**: Don't put secrets in code
2. **Use Key Vault**: Store all sensitive data as secrets
3. **Rotate Regularly**: Update secrets periodically
4. **Minimal Scope**: Only grant access to what's needed
5. **Never Log**: Don't log secret values

### ❌ Don't Do This
```python
# Hardcoded secret
api_key = "sk_live_123abc"

# Logged secret
logger.info(f"API key: {api_key}")

# Returned in response
return {"api_key": api_key}
```

### ✅ Do This
```python
# Get from Key Vault
api_key = secrets.get("api_key")

# Log without value
logger.info("Using API key from Key Vault")

# Don't return secret
return {"success": True}
```

## Error Handling

```python
@workflow(name="safe_secret_access")
async def safe_secret_access(context):
    try:
        api_key = secrets.get("api_key")
    except KeyError:
        return {
            "success": False,
            "error": "API key not configured",
            "action": "Add 'api_key' to Secrets in Settings"
        }

    # Use api_key...
```

## Updating Secrets

Update via UI:
1. **Settings** → **Secrets**
2. Find secret → Click **Edit**
3. Enter new value → **Save**

Or via workflow:
```python
secrets.set("api_key", "new_value")
```

<Aside type="tip">
**Pro Tip**: Use config references for easier management. Update secret once in Key Vault, all configs referencing it update automatically.
</Aside>

## Security Features

- **Encryption**: All secrets encrypted at rest
- **Access Control**: RBAC via Azure Key Vault
- **Audit Logs**: All access logged
- **No Exposure**: Secrets never returned in API responses
- **Org Isolation**: Complete separation between orgs
