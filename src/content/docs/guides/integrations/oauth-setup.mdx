OAuth 2.0 allows secure access to external APIs without storing passwords. Bifrost handles token refresh, storage, and multi-tenancy automatically.

## Quick Start

<Steps>

1. Create OAuth app in provider (Azure AD, GitHub, etc.)
2. Get Client ID, Client Secret, Tenant ID (if applicable)
3. Add OAuth connection in Bifrost
4. Authorize connection
5. Use in workflows

</Steps>

## Supported Providers

- **Microsoft Graph** (Azure AD)
- **GitHub**
- **Google** (coming soon)
- **Custom OAuth 2.0** providers

## Microsoft Graph Setup

### 1. Create App Registration

<Steps>

1. [Azure Portal](https://portal.azure.com) → **Azure AD** → **App registrations** → **+ New registration**

2. Configure:
   - Name: "Bifrost Integration"
   - Redirect URI: `https://your-domain.com/oauth/callback/microsoft-graph`

3. Copy **Application (client) ID** and **Tenant ID**

4. **Certificates & secrets** → **+ New client secret** → Copy value

5. **API permissions** → **+ Add permission** → **Microsoft Graph** → **Delegated permissions**
   - Add required scopes (e.g., `User.Read`, `User.ReadWrite.All`)
   - Click **Grant admin consent**

</Steps>

### 2. Add Connection in Bifrost

<Steps>

1. **Settings** → **Integrations** → **+ Add Connection**

2. Fill in:
   - Name: `microsoft-graph`
   - Provider: Microsoft Graph
   - Client ID: [from Azure]
   - Client Secret: [from Azure]
   - Tenant ID: [from Azure]
   - Scopes: `User.Read User.ReadWrite.All`

3. Click **Create**

</Steps>

### 3. Authorize Connection

<Steps>

1. Click connection → **Authorize**
2. Sign in with Microsoft
3. Consent to permissions
4. Redirected to Bifrost (status: Active)

</Steps>

## GitHub Setup

### 1. Create OAuth App

<Steps>

1. GitHub → **Settings** → **Developer settings** → **OAuth Apps** → **New OAuth App**

2. Configure:
   - Application name: "Bifrost"
   - Homepage URL: `https://your-domain.com`
   - Callback URL: `https://your-domain.com/oauth/callback/github`

3. Copy **Client ID** and **Client Secret**

</Steps>

### 2. Add in Bifrost

<Steps>

1. **Settings** → **Integrations** → **+ Add Connection**

2. Fill in:
   - Name: `github`
   - Provider: GitHub
   - Client ID: [from GitHub]
   - Client Secret: [from GitHub]
   - Scopes: `repo user`

3. **Create** → **Authorize** → Sign in with GitHub

</Steps>

## Use in Workflows

```python
from bifrost import workflow, oauth, ExecutionContext
import aiohttp

@workflow(name="get_user_info")
async def get_user_info(context: ExecutionContext, user_id: str):
    # Get OAuth credentials (auto-refreshes tokens)
    creds = oauth.get_oauth_connection("microsoft-graph")

    # Build request
    url = f"https://graph.microsoft.com/v1.0/users/{user_id}"
    headers = {"Authorization": creds.get_auth_header()}

    # Call API
    async with aiohttp.ClientSession() as session:
        async with session.get(url, headers=headers) as resp:
            if resp.status == 200:
                user = await resp.json()
                return {"success": True, "user": user}
            else:
                return {"success": False, "error": await resp.text()}
```

## Token Refresh

Bifrost automatically refreshes expired tokens:

```python
# No refresh logic needed - handled automatically
creds = oauth.get_oauth_connection("microsoft-graph")
auth_header = creds.get_auth_header()  # Always returns valid token
```

## Organization Scoping

Each organization maintains separate OAuth connections:

- Org A has `microsoft-graph` connection with their Azure AD
- Org B has `microsoft-graph` connection with their Azure AD
- Workflows automatically use correct org's credentials

```python
@workflow(name="example")
async def example(context):
    # Uses current org's OAuth connection automatically
    creds = oauth.get_oauth_connection("microsoft-graph")
```

## Multiple Connections

Create multiple connections to same provider:

```python
# Connection for production tenant
creds_prod = oauth.get_oauth_connection("microsoft-graph-prod")

# Connection for dev tenant
creds_dev = oauth.get_oauth_connection("microsoft-graph-dev")
```

## Error Handling

```python
from bifrost import oauth

@workflow(name="safe_oauth_call")
async def safe_oauth_call(context):
    try:
        creds = oauth.get_oauth_connection("microsoft-graph")
        # Use creds...

    except oauth.OAuthConnectionNotFound:
        return {
            "success": False,
            "error": "OAuth connection not configured",
            "action": "Set up Microsoft Graph OAuth in Settings"
        }

    except oauth.TokenRefreshFailed:
        return {
            "success": False,
            "error": "Token refresh failed",
            "action": "Reconnect OAuth connection"
        }
```

## Scopes Reference

### Microsoft Graph Common Scopes

**User Management**:
- `User.Read` - Read own profile
- `User.ReadWrite.All` - Read/write all users
- `User.ReadBasic.All` - Read basic info for all users

**Group Management**:
- `Group.Read.All` - Read all groups
- `Group.ReadWrite.All` - Read/write all groups

**Mail**:
- `Mail.Read` - Read user's mail
- `Mail.Send` - Send mail as user

**Calendar**:
- `Calendars.Read` - Read calendars
- `Calendars.ReadWrite` - Read/write calendars

### GitHub Scopes

- `repo` - Access repositories
- `user` - Read user profile
- `admin:org` - Manage organization
- `gist` - Manage gists

## Custom OAuth Provider

For providers not built-in:

<Steps>

1. **Settings** → **Integrations** → **+ Add Connection**

2. Select **Custom OAuth 2.0**

3. Provide:
   - Authorization URL
   - Token URL
   - Client ID
   - Client Secret
   - Scopes
   - Redirect URI

4. Follow provider's OAuth documentation

</Steps>

## Troubleshooting

### Connection Fails to Authorize

- Verify Redirect URI matches exactly
- Check Client ID and Secret are correct
- Ensure scopes are valid for provider
- Verify admin consent granted (if required)

### Token Refresh Fails

- Check if refresh token is valid
- Verify app permissions haven't changed
- Reconnect OAuth connection

### API Calls Return 401

- Token may be expired (should auto-refresh)
- Check if scopes include required permissions
- Verify OAuth connection is active

<Aside type="caution">
**Security**: Never expose Client Secrets. Bifrost stores them encrypted in Azure Key Vault.
</Aside>

## Best Practices

1. **Minimal Scopes**: Request only needed permissions
2. **Test Connections**: Verify OAuth works before using in production
3. **Monitor Expiration**: Set reminders for client secret renewal
4. **Use Org Scoping**: Separate OAuth per organization
5. **Handle Errors**: Always catch OAuth exceptions
