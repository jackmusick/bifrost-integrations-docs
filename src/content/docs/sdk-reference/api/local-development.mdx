## Development Environment Options

You have two options for local development:

<CardGrid>
  <Card title="Dev Container (Recommended)" icon="docker">
    Pre-configured environment with all dependencies

    **Pros:**
    - Consistent across all developers
    - No manual dependency installation
    - Works on Windows, Mac, Linux
    - Includes VS Code extensions

    **Best for:** Most developers, especially teams
  </Card>

  <Card title="Manual Setup" icon="laptop">
    Install dependencies directly on your machine

    **Pros:**
    - More control over versions
    - Faster startup (no container build)
    - Native performance

    **Best for:** Experienced developers with specific requirements
  </Card>
</CardGrid>

## Option 1: Dev Container Setup

The dev container provides a complete, reproducible development environment.

### Prerequisites

- **VS Code** with Dev Containers extension
- **Docker Desktop** (running)
- **Git** (for cloning the repository)

### Setup Steps

<Steps>

1. **Install Prerequisites**

   - [VS Code](https://code.visualstudio.com/)
   - [Docker Desktop](https://www.docker.com/products/docker-desktop/)
   - [Dev Containers extension](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers)

2. **Clone the Repository**

   ```bash
   git clone https://github.com/jackmusick/bifrost-api.git
   cd bifrost-api
   ```

3. **Open in VS Code**

   ```bash
   code .
   ```

4. **Reopen in Container**

   VS Code will detect the `.devcontainer` folder and prompt you to reopen in a container.

   Alternatively:
   - Press `Cmd+Shift+P` (Mac) or `Ctrl+Shift+P` (Windows/Linux)
   - Type "Dev Containers: Reopen in Container"
   - Press Enter

5. **Wait for Build**

   First time: ~2 minutes to build and configure
   Subsequent times: ~30 seconds

6. **Verify Installation**

   Once the container is running, open a terminal in VS Code and verify:

   ```bash
   python --version    # Should show 3.11+
   func --version      # Should show Azure Functions Core Tools 4.x
   azurite --version   # Should show Azurite version
   ```

7. **Start Development**

   ```bash
   func start
   ```

   The API will be available at http://localhost:7071

</Steps>

### What's Included

The dev container comes pre-configured with:

- **Python 3.11** with all dependencies from `requirements.txt`
- **Azure Functions Core Tools v4**
- **Azure CLI**
- **Azurite** (Azure Storage Emulator)
- **Git**
- **VS Code extensions:**
  - Python
  - Pylance
  - Azure Functions
  - Ruff (linter)
  - Pyright (type checker)

### Container Customization

You can customize the dev container by editing `.devcontainer/devcontainer.json`:

```json
{
  "name": "Bifrost API",
  "build": {
    "dockerfile": "Dockerfile"
  },
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-python.python",
        "ms-azuretools.vscode-azurefunctions"
      ]
    }
  }
}
```

<Aside type="tip">
Changes to `devcontainer.json` require rebuilding the container: `Cmd+Shift+P` → "Dev Containers: Rebuild Container"
</Aside>

## Option 2: Manual Setup

If you prefer to install dependencies directly on your machine:

### Prerequisites

- **Python 3.11+**
- **Node.js 18+** (for Azure Functions Core Tools)
- **Azure Functions Core Tools v4**
- **Azurite** (or Docker to run Azurite)

### Setup Steps

<Steps>

1. **Install Python 3.11+**

   <Tabs>
     <TabItem label="macOS">
       ```bash
       # Using Homebrew
       brew install python@3.11

       # Or using pyenv
       brew install pyenv
       pyenv install 3.11
       pyenv global 3.11
       ```
     </TabItem>
     <TabItem label="Windows">
       Download from [python.org](https://www.python.org/downloads/) or use:

       ```powershell
       # Using winget
       winget install Python.Python.3.11
       ```
     </TabItem>
     <TabItem label="Linux">
       ```bash
       # Ubuntu/Debian
       sudo apt update
       sudo apt install python3.11 python3.11-venv

       # Fedora/RHEL
       sudo dnf install python3.11
       ```
     </TabItem>
   </Tabs>

2. **Install Azure Functions Core Tools**

   <Tabs>
     <TabItem label="macOS">
       ```bash
       brew tap azure/functions
       brew install azure-functions-core-tools@4
       ```
     </TabItem>
     <TabItem label="Windows">
       ```powershell
       npm install -g azure-functions-core-tools@4 --unsafe-perm true
       ```
     </TabItem>
     <TabItem label="Linux">
       ```bash
       npm install -g azure-functions-core-tools@4 --unsafe-perm true
       ```
     </TabItem>
   </Tabs>

3. **Install Azurite**

   <Tabs>
     <TabItem label="npm (All Platforms)">
       ```bash
       npm install -g azurite
       ```
     </TabItem>
     <TabItem label="Docker">
       ```bash
       docker run -d -p 10000:10000 -p 10001:10001 -p 10002:10002 \
         --name azurite \
         mcr.microsoft.com/azure-storage/azurite
       ```
     </TabItem>
   </Tabs>

4. **Clone and Setup Repository**

   ```bash
   git clone https://github.com/jackmusick/bifrost-api.git
   cd bifrost-api

   # Create virtual environment
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate

   # Install dependencies
   pip install -r requirements.txt
   ```

5. **Configure Local Settings**

   ```bash
   cp local.settings.example.json local.settings.json
   ```

   Edit `local.settings.json` to customize settings if needed.

6. **Start Azurite**

   In a separate terminal:

   ```bash
   azurite --silent
   ```

   Or if using Docker, Azurite should already be running from step 3.

7. **Start Functions Runtime**

   ```bash
   func start
   ```

   The API will be available at http://localhost:7071

</Steps>

### Verifying Manual Setup

Verify all components are working:

```bash
# Check Python version
python --version  # Should be 3.11+

# Check Functions Core Tools
func --version  # Should be 4.x

# Check Azurite is running
curl http://127.0.0.1:10000  # Should return Azurite info

# Check API health
curl http://localhost:7071/api/health  # Should return 200 OK
```

## Configuration

### local.settings.json

This file contains configuration for local development:

```json
{
  "IsEncrypted": false,
  "Values": {
    "AzureWebJobsStorage": "UseDevelopmentStorage=true",
    "FUNCTIONS_WORKER_RUNTIME": "python",
    "FUNCTIONS_WORKER_PROCESS_COUNT": "10",
    "AzureWebJobsFeatureFlags": "EnableWorkerIndexing"
  }
}
```

<Aside type="caution">
Never commit `local.settings.json` to Git. It's in `.gitignore` by default.
</Aside>

#### Important Settings

- **`AzureWebJobsStorage`**: Connection string for Azure Storage (Azurite in local dev)
- **`FUNCTIONS_WORKER_PROCESS_COUNT`**: Number of worker processes (10 for local dev to prevent blocking)
- **`AzureWebJobsFeatureFlags`**: Feature flags for Functions runtime

<Aside type="tip" title="Why 10 Workers Locally?">
Setting `FUNCTIONS_WORKER_PROCESS_COUNT=10` prevents blocking when workflows are executing. Production Azure Functions auto-scales across multiple instances, so this setting is only needed locally.
</Aside>

### Environment Variables

You can add environment variables to `local.settings.json`:

```json
{
  "Values": {
    "MY_CUSTOM_VAR": "value"
  }
}
```

Access them in code:

```python
import os
my_var = os.environ.get("MY_CUSTOM_VAR")
```

## Hot Reload

The Functions runtime supports hot reload for most code changes:

- **Functions code** (handlers) - ✅ Hot reload
- **Shared modules** (business logic) - ✅ Hot reload
- **SDK changes** - ✅ Hot reload
- **Configuration changes** - ❌ Requires restart
- **Dependency changes** - ❌ Requires restart

When hot reload doesn't work, restart the Functions runtime:

```bash
# Press Ctrl+C to stop
func start  # Start again
```

## Troubleshooting

### Port 7071 Already in Use

```bash
# Find the process using port 7071
lsof -ti:7071

# Kill it
kill -9 $(lsof -ti:7071)
```

### Azurite Not Starting

```bash
# Check if Azurite is running
ps aux | grep azurite

# Kill all Azurite processes
pkill azurite

# Start fresh
azurite --silent &
```

### Import Errors

If you see "ModuleNotFoundError", ensure:

1. Virtual environment is activated: `source .venv/bin/activate`
2. Dependencies are installed: `pip install -r requirements.txt`
3. You're in the correct directory: `cd bifrost-api`

### Functions Runtime Won't Start

```bash
# Clear Functions cache
rm -rf ~/.azure-functions-core-tools

# Reinstall Functions Core Tools
npm install -g azure-functions-core-tools@4 --unsafe-perm true
```

## Performance Tips

### Speed Up Container Builds

Add frequently-used Python packages to the Dockerfile so they're cached:

```dockerfile
RUN pip install --no-cache-dir \
    azure-functions \
    azure-data-tables \
    pydantic \
    # Add more here
```

### Faster Test Runs

Run specific test files instead of the entire suite:

```bash
./test.sh tests/unit/  # Only unit tests
./test.sh tests/integration/platform/test_sdk_from_workflow.py  # One file
```

### Reduce Console Noise

Lower the Functions runtime log level in `host.json`:

```json
{
  "logging": {
    "logLevel": {
      "default": "Warning"
    }
  }
}
```
